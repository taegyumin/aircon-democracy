# 다른 LLM을 위한 검증·재수행 프롬프트

이 문서는 **다른 LLM이 이 작업(한국 대학교 강의실 데이터셋 구축)을 직접 다시 수행하면서 검증**하도록 보내는 프롬프트입니다. 그대로 복사해서 사용하세요.

---

## 작업 컨텍스트

당신은 **에어컨 민주주의(Aircon Democracy)** 라는 한국 무료 시민 서비스의 강의실 데이터셋을 검증·재수행합니다. 이 서비스는 공공 공간(지하철·버스·강의실 등)에서 "추워요/적당해요/더워요" 3택 투표를 모읍니다.

이전 LLM이 한국 4년제 대학교 ~108개의 건물 데이터를 큐레이션해서 `packages/core/src/universities/data/seoul.json` 과 `korea.json` 에 넣어 두었습니다. 사용자가 그 결과의 정확성·완전성을 못 미더워해서, 당신에게 **검증을 빙자한 재수행**을 시킵니다.

## 핵심 원칙

1. **추측 금지**. 공식 캠퍼스맵 / 위키백과 / 나무위키 / 학교 부속 페이지에서 **직접 확인된 건물만** 포함하세요.
2. **크로스체크**. 한 출처에만 나오는 건물은 의심하고, 가능하면 2개 이상 출처에서 일치하는 것만 채택.
3. **출처 보존**. 모든 항목의 `sources` 필드에 검증한 URL을 남기세요.
4. **빈 캠퍼스 OK**. 공개 데이터가 부족한 학교는 `buildings: []` 로 두고 사용자 freeform 입력으로 폴백.
5. **건물 수에 집착 X**. 1개 학교에 5개 건물이라도 정확하면 100개 건물보다 낫습니다.

## 절차

### Step 1: 기존 데이터셋 진단 (먼저 이거 하세요)

```bash
cd /Users/taegyumin/.claude/worktrees/seoul-univs  # 이미 체크아웃되어 있을 가능성
git log --oneline -10
cat packages/core/src/universities/data/seoul.json | jq '.schools | length'
cat packages/core/src/universities/data/korea.json | jq '.schools | length'
```

각 학교에 대해 다음을 검증:

| 검사 항목 | 도구 |
|---|---|
| `id` 가 unique 한가? | jq |
| `placeIdPrefix` 가 unique 한가? | jq |
| `sources` 가 ≥1개 URL 인가? | jq |
| `campuses[].district` 가 실제 행정구역인가? | 위키 확인 |
| 건물 `code` 가 캠퍼스 내에서 unique 한가? | jq |
| 건물명에 명백한 오타/잘못된 정보가 없는가? | sampling |

**100개 학교를 다 검증하지 마세요.** 무작위 표본 ~15개 학교만 깊게 보고, 나머지는 위 schema 검사만 자동화하세요.

### Step 2: 표본 검증 시 사용할 방법

**무작위 15개 학교 선정** → 각 학교의 공식 캠퍼스맵 URL을 직접 fetch → 건물 명단을 비교.

발견 가능한 문제 패턴:
- **건물 이름 오타**: "백남학술정보관" → 실제로는 "백남관" 일 수도
- **유효하지 않은 건물**: 옛 이름인데 현 건물이 아닌 경우 (예: 동국대 신공학관이 실제로는 다른 동)
- **번호 체계 혼동**: SNU "관악 301동" vs "신공학관 301호" 같은 헷갈림
- **잘못된 campus 분류**: 단국대 죽전 건물을 천안 캠퍼스에 넣은 경우
- **누락된 메이저 건물**: 학생들이 흔히 가는 건물인데 없음

### Step 3: 본격적인 재수행 (가장 중요)

기존 데이터를 **참고만 하고**, 다음 10개 학교를 **처음부터 다시** 큐레이션하세요:

1. 서울대학교 (이건 IST 무선현황 PDF 기반 — 이미 정밀, 검증만)
2. 연세대학교 신촌
3. 고려대학교 안암
4. 한양대학교 서울
5. 동국대학교 서울 (이미 1~36 완벽한 list)
6. 부산대학교
7. 경북대학교
8. KAIST 대전 본원
9. POSTECH 포항
10. 본인이 가장 잘 아는 학교 1개 (지인이 다니거나)

각각에 대해:
- 공식 사이트 + 위키 동시 확인
- 최소 10개 검증된 건물
- 단과대(college) 매핑이 맞는지
- 동번호 체계가 일관되는지

### Step 4: 누락된 학교 추가 시도

서울·전국 4년제 중에 빠진 학교를 더 찾아보세요. 가능성 있는 것들:

## 추가 컨텍스트 (도움될 수 있는 사실)

- 이전 LLM이 가장 의지한 출처: 나무위키 (한국 대학 캠퍼스 페이지가 학생들이 직접 정리해서 풍부함). 단 namu.wiki 직접 fetch 시 403 자주 발생 — 우회 위해 Web search 결과의 snippet을 사용했음.
- 이전 LLM이 못한 것: 공식 PDF가 raster image-only 일 때 OCR 불가. 대부분 PDF가 그랬음.
- 이전 LLM이 발견한 핵심 데이터: **SNU IST 무선현황 PDF** (`https://ist.snu.ac.kr/wp-content/uploads/sites/186/2024/02/%EB%AC%B4%EC%84%A0%ED%98%84%ED%99%A9_202402.pdf`) — 이게 한국 대학 중 유일하게 캠퍼스/동번호/호실/호실명 정형 데이터를 공개함. 다른 학교는 비슷한 인벤토리 없음.
- 인증 필요 시스템: **성균관대 classinfo.skku.edu** — 학생용 강의실 시스템이지만 로그인 강제. 협조자 있으면 SNU급 데이터 확보 가능.
- 이전 LLM이 의도적으로 제외한 것: **육군사관학교** (보안), **사이버대학** (출석 수업 없음 — 강의실 데이터 의미 없음).

## 작업 범위 한정

이 작업의 목표는 "**한국의 모든 대학 모든 건물 데이터베이스 완성**"이 아닙니다. **각 학교에서 학생들이 흔히 가는 메인 건물 5~30개만 정확히 등록**하는 게 목표입니다. 단과대 건물·도서관·학생회관·본관 정도가 핵심. 기숙사·연구센터·부속병원은 우선순위 낮음.
